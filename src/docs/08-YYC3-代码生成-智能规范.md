#### 2.1.2 意图匹配函数

```typescript
/**
 * @function getResponseForInput
 * @description 关键词匹配 → 路由到对应响应模板
 * @param input 用户输入文本
 * @returns 完整响应文本（Markdown 格式）
 */
function getResponseForInput(input: string): string {
  const lower = input.toLowerCase();
  if (lower.includes('代码') || lower.includes('code') || lower.includes('/code'))
    return responses.code;
  if (lower.includes('架构') || lower.includes('arch') || lower.includes('/arch'))
    return responses.architecture;
  if (lower.includes('帮助') || lower.includes('help') || lower.includes('/help'))
    return responses.help;
  return responses.default;
}
```

#### 2.1.3 意图匹配优先级

| 优先级 | 关键词                  | 路由目标                 | 响应类型                         |
| ------ | ----------------------- | ------------------------ | -------------------------------- |
| 1      | `/code`, `代码`, `code` | `responses.code`         | 代码生成（含 TypeScript 代码块） |
| 2      | `/arch`, `架构`, `arch` | `responses.architecture` | 架构建议（含表格和列表）         |
| 3      | `/help`, `帮助`, `help` | `responses.help`         | 快捷指令帮助文档                 |
| 4      | 其他所有输入            | `responses.default`      | 通用 AI 助手问候                 |

### 2.2 流式响应引擎

#### 2.2.1 核心流式函数

```typescript
// /src/app/utils/ai-simulator.ts

/**
 * @function simulateStreamResponse
 * @description 模拟 AI 流式响应（逐字符输出，模拟 SSE）
 * @param input     用户输入
 * @param onChunk   每次输出 1-3 个字符的回调
 * @param onComplete 输出完成的回调
 */
export async function simulateStreamResponse(
  input: string,
  onChunk: (chunk: string) => void,
  onComplete: () => void
): Promise<void> {
  const fullResponse = getResponseForInput(input);
  const chars = [...fullResponse]; // 支持 Unicode（中文安全）
  let index = 0;

  return new Promise((resolve) => {
    const interval = setInterval(() => {
      if (index < chars.length) {
        // 随机 1-3 字符块，模拟真实 LLM 输出节奏
        const chunkSize = Math.floor(Math.random() * 3) + 1;
        const chunk = chars.slice(index, index + chunkSize).join('');
        onChunk(chunk);
        index += chunkSize;
      } else {
        clearInterval(interval);
        onComplete();
        resolve();
      }
    }, 20); // 20ms 间隔 ≈ 50 chunks/sec
  });
}
```

#### 2.2.2 流式参数设计

| 参数               | 值             | 说明                                     |
| ------------------ | -------------- | ---------------------------------------- |
| 间隔 (interval)    | 20ms           | 每 20ms 发送一个 chunk                   |
| 块大小 (chunkSize) | 1-3 字符       | 随机化模拟真实 LLM 输出                  |
| Unicode 安全       | `[...string]`  | 使用展开运算符正确处理多字节字符         |
| 完成回调           | `onComplete()` | 流结束后清除 interval 并 resolve Promise |

#### 2.2.3 ChatInterface 流式集成

```typescript
// /src/app/components/ChatInterface.tsx（概念示意）

// /src/app/components/CodeEditor.tsx（概念示意）

export function CodeEditor() {
  const pendingCodeInjection = useAppStore((state) => state.pendingCodeInjection);
  const clearCodeInjection = useAppStore((state) => state.clearCodeInjection);

  useEffect(() => {
    if (pendingCodeInjection) {
      // 切换到注入的文件
      setActiveFile(pendingCodeInjection.filename);
      // 显示 Diff 预览
      showDiffPreview(pendingCodeInjection.code);
    }
  }, [pendingCodeInjection]);

  const handleAcceptChanges = () => {
    if (pendingCodeInjection) {
      applyCode(pendingCodeInjection.code);
      clearCodeInjection();
    }
  };

  const handleRejectChanges = () => {
    clearCodeInjection();
  };
}

### 3.4 注入安全检查

| 检查项       | 规则                                                  | 处理方式           |
| ------------ | ----------------------------------------------------- | ------------------ |
| 文件名验证   | 必须匹配 `/^[\w\-./]+\.(tsx?\|jsx?\|css\|json\|md)$/` | 拒绝非法文件名     |
| 代码大小限制 | 单次注入不超过 100KB                                  | 超限截断并提示     |
| 语言匹配     | `language` 必须匹配 Monaco 支持的语言                 | 回退到 `plaintext` |
| 重复注入防护 | 相同 `filename + code` 不重复触发                     | 忽略重复注入       |

/**
 * @function convertStyleToTailwind
 * @description 将 ComponentStyle 转换为 Tailwind CSS 类字符串
 */
function convertStyleToTailwind(style: ComponentStyle): string {
  const classes: string[] = [];

  // 布局
  if (style.display === 'flex') classes.push('flex');
  if (style.flexDirection === 'column') classes.push('flex-col');
  if (style.justifyContent === 'center') classes.push('justify-center');
  if (style.alignItems === 'center') classes.push('items-center');

  // 间距 (基于 4px 栅格)
  if (typeof style.padding === 'number') {
    classes.push(`p-${style.padding / 4}`);
  }
  if (typeof style.gap === 'number') {
    classes.push(`gap-${style.gap / 4}`);
  }

  // 圆角
  if (typeof style.borderRadius === 'number') {
    const radiusMap: Record<number, string> = {
      4: 'rounded-sm',
      8: 'rounded',
      12: 'rounded-md',
      16: 'rounded-lg',
      24: 'rounded-xl',
      9999: 'rounded-full',
    };
    classes.push(radiusMap[style.borderRadius] || `rounded-[${style.borderRadius}px]`);
  }

  return classes.join(' ');
}
```

## 8. 智能辅助闭环

### 8.1 AI 辅助工作流

```
用户操作触发
    ↓
上下文收集 (当前文件、选中代码、Design JSON 状态)
    ↓
AI 意图理解 (Slash Command 或自然语言分析)
    ↓
流式响应生成 (simulateStreamResponse)
    ↓
代码块提取 & 展示
    ↓
用户选择: 接受 / 拒绝 / 修改
    ↓
代码注入到编辑器 (injectCode → pendingCodeInjection)
    ↓
Diff 预览 → 确认应用
    ↓
文件系统更新 → 预览刷新
    ↓
效果反馈 → 继续对话
    ↓
（循环）
```

### 8.2 上下文收集策略

| 上下文类型 | 数据源               | 用途                      |
| ---------- | -------------------- | ------------------------- |
| 当前文件   | `store.selectedFile` | 确定代码注入目标          |
| 文件内容   | `FileManager` 文件树 | 提供代码上下文            |
| 对话历史   | `store.messages[]`   | 维持多轮对话连续性        |
| 面板状态   | `store.designRoot`   | 理解当前布局              |
| 主题模式   | `store.theme`        | 生成适配当前主题的代码    |
| 语言设置   | `store.language`     | 决定 AI 响应语言（zh/en） |

### 8.3 错误诊断闭环

```
代码变更
    ↓
Monaco Editor 实时检测
    ↓
TypeScript 类型错误 / ESLint 规则违反
    ↓
错误标记 (红色波浪线 + 错误面板)
    ↓
用户触发 AI 诊断 (点击 "AI 修复" 或描述问题)
    ↓
AI 分析错误上下文
    ↓
生成修复建议 (代码块)
    ↓
用户确认 → 代码注入 → 错误消除
    ↓
（循环）
```

## 10. 代码生成质量保障

### 10.1 生成代码规范检查

| 检查项          | 规则                             | 工具              |
| --------------- | -------------------------------- | ----------------- |
| TypeScript 类型 | `strict: true`, 无 `any` 滥用    | Monaco TS 诊断    |

| 变量名   | camelCase        | `streamingContent`, `pendingCodeInjection`      |
