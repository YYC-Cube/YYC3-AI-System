### 1.3 Layer 4: 功能逻辑层

**职责**: 路由决策、面板管理、组件系统、全局状态管理

#### 路由系统

```typescript
// /src/app/routes.ts
import { createBrowserRouter } from 'react-router';
import { HomePage } from './components/HomePage';
import { IDELayout } from './components/IDELayout';

export const router = createBrowserRouter([
  { path: '/', Component: HomePage },
  { path: '/ide', Component: IDELayout },
]);
```

**路由决策逻辑**:

- `HomePage` 智能路由：分析用户首次聊天内容语义 -> 检测编程意图 -> 自动 `navigate('/ide')`
- `IDELayout` 视图模式：`viewMode` 状态切换 `code` / `preview` / `fullscreen`

#### 面板管理系统

```typescript
// Zustand Store 中的面板映射 (DnD 可交换)
panelMap: { left: 'chat', middle: 'files', right: 'code' }

// 交换逻辑
swapPanels: (from, to) => {
  const next = { ...state.panelMap }
  const temp = next[from]
  next[from] = next[to]
  next[to] = temp
  return { panelMap: next }
}
```

**面板内容映射**:

| Slot     | 默认内容 | 组件          | 工具栏        |
| -------- | -------- | ------------- | ------------- |
| `left`   | `chat`   | ChatInterface | LeftToolbar   |
| `middle` | `files`  | FileManager   | MiddleToolbar |
| `right`  | `code`   | CodeEditor    | RightToolbar  |

### 1.4 Layer 3: AI 智能层

**职责**: 意图识别、代码生成、SSE 流式通信、代码注入

#### SSE 流式引擎架构

```
ChatInterface
    |
    | callModelAPIStream(model, messages, onChunk, onDone, onError)
    |
    +--[provider === 'openai']
    |    POST {endpoint}/v1/chat/completions
    |    Headers: { Authorization: 'Bearer {apiKey}' }
    |    Body: { model, messages, stream: true }
    |    Parse: data: {"choices":[{"delta":{"content":"..."}}]}
    |
    +--[provider === 'claude']
    |    POST {endpoint}/v1/messages
    |    Headers: { x-api-key: {apiKey}, anthropic-version: '2023-06-01' }
    |    Body: { model, messages, stream: true }
    |    Parse: event: content_block_delta + data: {"delta":{"text":"..."}}
    |
    +--[provider === 'ollama']
         POST {endpoint}/api/chat
         Body: { model, messages, stream: true }
         Parse: NDJSON {"message":{"content":"..."}}
```

#### 代码注入 Diff 管线

````
1. ChatInterface 渲染 AI 响应 Markdown
2. 代码块识别 (```language ... ```)
3. 用户点击「应用到编辑器」按钮
4. handleApplyCode(code, language)
     |-- 语言 -> 文件名映射 (tsx->ChatInterface.tsx 等)
     |-- injectCode(filename, code, language) -> Store
5. CodeEditor useEffect 检测 pendingCodeInjection
     |-- setDiffOriginal(原始内容)
     |-- setDiffModified(注入内容)
     |-- setActiveView('diff')
6. Monaco DiffEditor 并排渲染
7. 用户 [接受变更] -> FILE_CONTENTS[filename] = modified
       [拒绝]     -> setActiveView('code')
````

### 1.5 Layer 2: 数据协同层

**职责**: CRDT 文档管理、光标感知协议、数据持久化

#### yjs CRDT 架构

```
CollaborationManager (Singleton)
    |
    +-- Y.Doc
    |    |-- getText('ChatInterface.tsx') -> Y.Text
    |    |-- getText('App.tsx')           -> Y.Text
    |    |-- getText('store.ts')          -> Y.Text
    |    |-- getText('types.ts')          -> Y.Text
    |    |-- getText('routes.ts')         -> Y.Text
    |    |
    |    +-- Y.Doc.on('update', callback)  // CRDT 增量更新
    |    +-- Y.applyUpdate(doc, update)    // 远程更新应用
    |
    +-- SimpleAwareness
    |    |-- states: Map<clientID, { user: CollabUser }>
    |    |-- setLocalState({ user })       // 本地用户状态
    |    |-- setRemoteState(id, { user })  // 远程用户状态
    |    |-- on('change', handler)         // 状态变更监听
    |
    +-- SimulatedPeerEngine
         |-- 3 个模拟用户: Alice / Bob / Carol
         |-- 光标移动定时器 (3000ms)
         |-- 在线切换定时器 (20000ms)
         |-- 编辑活动定时器 (5000ms)
```

**CRDT 合并原理**:

```
   User A (ClientID: 100)        User B (ClientID: 200)
          |                              |
   Insert "hello" at pos 5       Insert "world" at pos 5
          |                              |
          v                              v
   Y.Text: "...hello..." (v1a)   Y.Text: "...world..." (v1b)
          |                              |
          +-------> CRDT Merge <---------+
                       |
                       v
            Y.Text: "...helloworld..." (v2)
            (确定性排序: clientID 小的在前)
```

## 2. 前端技术栈详解

### 2.1 核心框架

| 技术         | 实际版本 | 用途     | 选型理由                                    |

| 维度        | React 18                 | Vue 3        | Svelte 5 | Solid |
| ----------- | ------------------------ | ------------ | -------- | ----- |
| 生态规模    | **Top 1**                | Top 2        | 成长中   | 小众  |

| 招聘市场    | **最大**                 | 大           | 小       | 极小  |

**结论**: React 在 IDE 类应用的生态成熟度上无可替代，特别是 Monaco Editor 和 yjs 的 React 绑定最为完善。

#### Zustand 选型论证 (vs Redux / Jotai / MobX)

| 维度           | Zustand 5      | Redux Toolkit     | Jotai  | MobX   |

| 包体积         | **1.1KB**      | 10KB+             | 3KB    | 15KB+  |

**结论**: Zustand 的 `create` + `persist` + `selector` 三件套完美匹配 IDE 应用的状态需求，零样板代码，内建 localStorage 持久化。

## 3. 主题引擎架构

### 3.1 Token 系统设计

**核心文件**: `/src/app/utils/theme.ts`

**设计原则**: 所有组件必须通过 `getThemeTokens(theme)` 获取颜色 token，禁止硬编码。

#### Token 分类架构

## 7. CI/CD 流水线设计

### 7.1 GitHub Actions 流水线

```yaml
# .github/workflows/ci.yml
name: YYC3 CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  # Stage 1: 代码质量检查
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint # ESLint + TypeScript
      - run: pnpm type-check # tsc --noEmit

  # Stage 2: 单元测试
  test:
    runs-on: ubuntu-latest
    needs: lint
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - run: pnpm install --frozen-lockfile
      - run: pnpm test # Vitest
      - run: pnpm test:coverage # 覆盖率报告

  # Stage 3: 构建
  build:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - run: pnpm install --frozen-lockfile
      - run: pnpm build # vite build
      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/

  # Stage 4: 部署 (main branch only)
  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/download-artifact@v4
      - name: Deploy to Production
        run: echo "Deploy to Vercel/Cloudflare"
```
