# 01 - YYC3 机制总结 - 核心设计

## 1. 品牌理念体系

### 1.1 品牌标识

**YYC3 = YanYuCloudCube**

| 维度 | 中文 | English | 内涵 |
|------|------|---------|------|
| 品牌全称 | 言语云立方 | YanYuCloudCube | 语言 + 云端 + 立方体(多维空间) |
| 核心理念 | 言启象限 · 语枢未来 | Words Initiate Quadrants, Language Serves as Core for Future | 语言开启创新象限，驱动未来智能 |
| 愿景宣言 | 万象归元于云枢 · 深栈智启新纪元 | All things converge in cloud pivot; Deep stacks ignite a new era of intelligence | 万物汇聚云端，深度技术栈开启新时代 |

### 1.2 核心使命

**言传千行代码 | 语枢万物智能**

通过自然语言传递代码逻辑，以语言中枢连接万物智能，实现人机协作的全新范式。

### 1.3 产品定位

**YYC3 PortAISys** (Portable Intelligent AI System) — 多联式低码设计平台

- **设计即代码**: 将设计师的视觉设计直接转化为可运行的生产级代码
- **实时预览**: 在每次设计变更时立即提供实时预览反馈
- **多联式布局**: 支持自由拖拽、合并、拆分的多面板布局系统
- **智能辅助**: 通过 AI 提供属性建议、代码片段、错误诊断
- **配置即部署**: 生成的代码可直接部署到生产环境

## 3. 核心功能机制 (已实施)

### 3.1 视觉风格体系 - Liquid Glass 拟物毛玻璃

**主题引擎**: `/src/app/utils/theme.ts` 中的 `getThemeTokens()` 提供完整 Token 体系

| Token 分类 | 说明 | 示例 |
|-----------|------|------|
| `surface.*` | 表面/背景色 | `surface.glass`, `surface.toolbar`, `surface.inset` |
| `border.*` | 边框色 | `border.subtle`, `border.medium`, `border.popover` |
| `text.*` | 文本色 | `text.primary`, `text.secondary`, `text.muted`, `text.dimmed` |
| `interactive.*` | 交互元素色 | `interactive.icon`, `interactive.iconBtn` |
| `syntax.*` | 代码语法高亮 | `syntax.keyword`, `syntax.string`, `syntax.comment` |
| `accent.*` | 品牌强调色 | `accent.primary`, `accent.link`, `accent.activeBg` |
| `status.*` | 状态色 | `status.error`, `status.warning`, `status.success` |
| `codeBlock.*` | 代码块样式 | `codeBlock.bg`, `codeBlock.inlineClass` |

**5 种主题模式**:

| 主题 ID | 中文名 | 基调 | Accent 色 |
|---------|--------|------|-----------|
| `light` | 日光 | 亮色 | `#6366f1` (indigo) |
| `dark` | 暗夜 | 暗色 | `#818cf8` (indigo) |
| `midnight` | 午夜蓝 | 暗色 | `#60a5fa` (blue) |
| `forest` | 森林绿 | 暗色 | `#34d399` (emerald) |
| `sunset` | 日落橙 | 亮色 | `#fb923c` (orange) |

**规则**: 所有组件必须通过 token 驱动而非硬编码颜色。

### 3.3 智能路由决策系统

**路由表** (`/src/app/routes.ts`):

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | `HomePage` | 首页入口，品牌标识 + AI 聊天框 + 最近项目 |
| `/ide` | `IDELayout` | 多联式布局设计器，三栏式 IDE 工作台 |

**决策逻辑**:

- **B. 智能 AI 交互工作台**: 持续监控用户交流内容，识别深度编程需求后切换至全屏交互模式

### 3.7 增量 Diff + Merge

**触发链路**: ChatInterface 代码块「应用到编辑器」按钮 -> `injectCode()` -> CodeEditor `activeView: 'diff'`

**Diff 工具栏**:
- `GitCompare` 图标 + 文件名 + "Diff 预览" 标签
- **接受变更** (`CheckCheck`): 将 modified 内容写入 `FILE_CONTENTS[filename]` + 绿色 toast
- **拒绝** (`XCircle`): 直接回到代码视图
- **关闭** (`X`): 取消 diff 视图

**Monaco DiffEditor 配置**:
- `renderSideBySide: true` (并排对比)
- `readOnly: true` (仅查看)
- 左侧原始代码 / 右侧注入代码
- 行级增删高亮（绿色新增 / 红色删除）

### 4.1 Zustand Store 结构

**位置**: `/src/app/store.ts`

**持久化**: `persist` 中间件 -> `localStorage('yyc3-storage')`

```typescript
interface AppState {
  // ── 主题 ──
  theme: ThemeMode                    // 5 种主题模式
  customThemeConfig: CustomThemeConfig // 自定义主题配置

  // ── 视图 ──
  viewMode: ViewMode                  // 'code' | 'preview' | 'fullscreen'
  selectedFile: string | null         // 当前编辑文件

  // ── 布局 ──
  panelMap: { left, middle, right }   // 面板内容映射(DnD 可交换)
  terminalVisible: boolean            // 终端可见性
  terminalHeight: number              // 终端高度

  // ── 消息 ──
  messages: Message[]                 // AI 对话历史

  // ── 项目 ──
  recentProjects: ProjectMeta[]       // 最近项目列表
  designRoot: DesignRoot              // Design JSON 根节点

  // ── AI 模型 ──
  aiModels: AIModel[]                 // 配置的 AI 模型列表
  activeModelId: string | null        // 当前活跃模型
  modelSettingsOpen: boolean          // 模型设置面板开关

  // ── 协同编辑 (yjs 驱动) ──
  collaborators: CollabUser[]         // 远程协作者列表(awareness 同步)

  // ── 代码注入 ──
  pendingCodeInjection: {             // AI -> CodeEditor 注入通道
    filename: string
    code: string
    language: string
  } | null
}
```

## 13. 最佳实践

### 开发规范

1. **Token 驱动**: 所有颜色通过 `getThemeTokens()` 获取，禁止硬编码
2. **组件单一职责**: 每个组件专注一个功能域
3. **状态最小化**: 只在 Zustand 中存储必要的共享状态
4. **类型安全**: 所有 props 和 state 定义 TypeScript 接口
5. **文档同步**: 代码变更时同步更新对应文档

### 性能最佳实践

1. **懒加载**: SyntaxHighlighter 等重依赖使用动态 import
2. **缓存计算**: useMemo 缓存诊断统计、折叠区域等
3. **精细订阅**: Zustand selector 避免全量重渲染
4. **布局稳定**: react-resizable-panels 避免 Layout Shift
5. **防抖输入**: 搜索和终端输入使用防抖处理

### 安全最佳实践

1. **API Key 安全**: 仅在客户端 localStorage 中，不上传服务器
2. **输入验证**: Zod schema 验证用户输入
3. **XSS 防护**: ReactMarkdown 安全渲染
4. **最小权限**: 组件只访问所需的 store slice
5. **依赖审计**: 定期更新 npm 依赖
