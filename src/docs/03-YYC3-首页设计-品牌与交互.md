## 5. 智能路由决策系统

### 5.1 路由决策流程

```
用户在首页 AI 聊天框输入消息
    |
    v
handleSend() 触发
    |
    +-- addMessage({ role: 'user', content: input.trim() })
    |   // 将用户消息写入 Zustand Store
    |
    +-- setInput('')
    |   // 清空输入框
    |
    +-- navigate('/ide')
        // 跳转到 IDE 多联式布局设计器
        // 用户需求上下文通过 Store.messages 传递
```

### 5.2 决策路径 A: 多联式布局设计器

**当前实现**: 首页输入的所有内容都导航到 `/ide`

**规划增强** (意图分析):

```typescript
// 未来: 基于 NLP 的意图分析路由
const analyzeIntent = (input: string): RouteDecision => {
  const programmingKeywords = ['代码', '组件', '函数', '页面', '接口', 'API',
    'code', 'component', 'function', 'page', 'create', 'build']
  const designKeywords = ['设计', '布局', '样式', 'UI', '界面', '颜色',
    'design', 'layout', 'style', 'color']

  const hasProgramming = programmingKeywords.some(k =>
    input.toLowerCase().includes(k))
  const hasDesign = designKeywords.some(k =>
    input.toLowerCase().includes(k))

  if (hasProgramming || hasDesign) {
    return { target: '/ide', mode: 'code' }
  }
  return { target: '/ide', mode: 'chat' }
}
```

### 5.3 决策路径 B: 智能 AI 交互工作台

**触发条件** (规划):
- 识别为深度对话需求 (非编程相关)
- 检测到复杂问答场景
- 需要全屏交互模式

**规划实现**: `/ide` 页面内的 `viewMode: 'fullscreen'` 切换

### 5.4 上下文传递机制

```
HomePage                    Zustand Store               IDELayout
    |                           |                          |
    | addMessage(userMsg)       |                          |
    |-------------------------->|                          |
    |                           | messages: [..., userMsg] |
    | navigate('/ide')          |                          |
    |                           |                          |
    |                           |<-- 读取 messages --------|
    |                           |                          |
    |                           | ChatInterface 显示消息   |
    |                           | 自动发送到 AI 模型       |
```

**关键**: 上下文通过 Zustand Store 的 `messages` 数组传递，而非 URL 参数，确保复杂内容(多行文本、代码片段)的完整性。

}

## 7. 入场动画体系

### 7.1 Motion 动画序列

所有主内容区采用 **渐入 + 上移** 的序列动画:

| 元素 | delay | 初始状态 | 目标状态 | duration |
|------|-------|---------|---------|----------|
| 品牌标识区 | 0ms | `opacity:0, y:20` | `opacity:1, y:0` | 600ms |
| AI 聊天框 | 150ms | `opacity:0, y:20` | `opacity:1, y:0` | 600ms |
| 最近项目区 | 300ms | `opacity:0, y:20` | `opacity:1, y:0` | 600ms |

```typescript
<motion.div
  initial={{ opacity: 0, y: 20 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.6, delay: 0.15 }}  // 150ms 延迟
>
```

### 7.2 Peel 转场动画

**定义位置**: `App.tsx <style>` 标签

```css
@keyframes peelReveal {
  0%   { clip-path: circle(0%);   opacity: 1;   }
  60%  { clip-path: circle(100%); opacity: 0.6; }
  100% { clip-path: circle(150%); opacity: 0;   }
}
```

**触发机制**:

```typescript
// Store 中的 peelTransition 状态
peelTransition: { active: boolean; x: number; y: number } | null
triggerPeelTransition: (x: number, y: number) => void
```

**效果**: 从点击位置向外扩展的圆形揭示动画，用于页面切换场景。

## 10. 性能优化

### 10.1 动画性能

| 策略 | 实现 | 效果 |
|------|------|------|
| GPU 合成 | motion `transform` + `opacity` | 避免 layout 重排 |
| `will-change` | motion 自动管理 | 提示浏览器预优化 |
| `pointer-events-none` | 背景 Blob | 避免事件穿透 |
| `AnimatePresence` | 条件渲染动画 | 优雅的进出场 |

### 10.2 渲染优化

| 策略 | 实现 | 效果 |
|------|------|------|
| Zustand selector | `useAppStore(s => s.theme)` | 精细化订阅，避免全量渲染 |
| 条件渲染 | `showCommands && <CommandPanel/>` | 按需创建 DOM |
| 事件委托 | 项目卡片 `stopPropagation` | 避免事件冒泡冲突 |
| 静态提升 | `SLASH_COMMANDS`/`TOOL_ICONS` 模块级常量 | 避免重复创建 |

### 10.3 资源优化

| 资源 | 策略 | 说明 |
|------|------|------|
| Logo 图片 | `figma:asset` 引入 | Vite 自动优化 |
| Lucide 图标 | Tree-shaking | 仅打包使用的图标 |
| ThemeCustomizer | 按需渲染 | 模态窗口条件挂载 |
