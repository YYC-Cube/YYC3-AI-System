# 05 - YYC3 图标系统 - 设计规范

## 1. 图标设计原则

### 1.1 核心原则

| 原则         | 说明                       | 实践                                      |
| ------------ | -------------------------- | ----------------------------------------- |
| **一致性**   | 所有图标遵循统一的设计语言 | 全量使用 Lucide React，不混用其他图标库   |
| **清晰性**   | 图标含义明确，易于理解     | 每个图标附中/英文 `title` 和 `aria-label` |
| **简洁性**   | 避免过度装饰，保持简洁     | Lucide 原生 2px stroke、圆角              |
| **可识别性** | 在不同尺寸下都清晰可辨     | 统一 3-4 个尺寸层级                       |
| **无障碍性** | 考虑色盲用户和屏幕阅读器   | 语义颜色 + ARIA + title                   |

### 1.2 视觉风格

- **线条风格**: Lucide stroke-only 线性图标
- **线条粗细**: `stroke-width="2"` (Lucide 默认)
- **端点处理**: `stroke-linecap="round"` (圆角)
- **连接处理**: `stroke-linejoin="round"` (圆角)
- **填充**: 无填充 (纯线条)

---

## 2. 图标库规范

### 2.1 主图标库

**名称**: Lucide React
**包名**: `lucide-react`
**许可**: MIT
**风格**: 统一线性图标，源自 Feather Icons 扩展

**选择理由**:

- TypeScript 原生类型支持
- Tree-shaking 友好（仅打包实际导入的图标）
- 风格高度统一（2px stroke、round cap/join）
- 持续维护，社区活跃
- 与 Liquid Glass 毛玻璃风格视觉契合

### 2.2 导入方式

```typescript
// 按需导入（推荐）— Tree-shaking 只打包使用的图标
import { Send, Plus, Code, Sparkles } from 'lucide-react'

// 使用
<Send className="w-4 h-4" />
```

**禁止**: 不使用 `import * as Icons from 'lucide-react'` 全量导入

### 2.3 Tailwind CSS 尺寸控制

YYC3 项目中统一使用 Tailwind `w-*` / `h-*` 类控制图标尺寸，**不使用** Lucide 的 `size` prop:

```typescript
// 正确 ✅
<Send className="w-4 h-4" />

// 避免 ❌
<Send size={16} />
```

## 3. 尺寸体系

### 3.1 标准尺寸层级

| 层级     | Tailwind 类         | 像素 | 使用场景                                               |
| -------- | ------------------- | ---- | ------------------------------------------------------ |
| **XS**   | `w-2 h-2`           | 8px  | 状态指示圆点（非图标，用 `div`）                       |
| **SM**   | `w-3 h-3`           | 12px | ChevronDown 辅助箭头                                   |
| **MD**   | `w-3.5 h-3.5`       | 14px | Toolbar 图标（LeftToolbar/MiddleToolbar/RightToolbar） |
| **BASE** | `w-4 h-4`           | 16px | 标准功能图标（聊天框工具栏、发送按钮、面板内图标）     |
| **LG**   | `w-[18px] h-[18px]` | 18px | Header 导航图标（HomePage + Header）                   |
| **XL**   | `w-5 h-5`           | 20px | 用户面板头像内图标、强调图标                           |

### 3.2 组件-尺寸映射

| 组件               | 图标尺寸                   | 说明                 |
| ------------------ | -------------------------- | -------------------- |
| Header (IDE)       | `w-4 h-4` (16px)           | 导航栏功能图标       |
| Header (Home)      | `w-[18px] h-[18px]` (18px) | 首页导航稍大         |
| LeftToolbar        | `w-3.5 h-3.5` (14px)       | 工具栏功能按钮       |
| MiddleToolbar      | `w-3.5 h-3.5` / `w-4 h-4`  | 视图切换 / 返回按钮  |
| RightToolbar       | `w-3.5 h-3.5` (14px)       | 面板快捷按钮         |
| ChatInterface      | `w-4 h-4` (16px)           | 工具按钮、消息内图标 |
| FileManager        | `w-3.5 h-3.5` (14px)       | 文件树图标           |
| CodeEditor         | `w-3 h-3` ~ `w-4 h-4`      | 诊断标记、操作按钮   |
| IntegratedTerminal | `w-3.5 h-3.5` (14px)       | 终端标签操作         |
| ModelSettings      | `w-4 h-4` ~ `w-5 h-5`      | 模态面板内图标       |
| ThemeCustomizer    | `w-4 h-4` (16px)           | 设置面板图标         |
| PreviewPanel       | `w-3.5 h-3.5` ~ `w-4 h-4`  | 设备切换、地址栏     |

### 3.3 Bot 图标容器规范

AI 相关 Bot 图标使用固定容器 + 内图标的组合:

```typescript
// LeftToolbar: 20x20 容器 + 12x12 图标
<div className="w-5 h-5 rounded flex items-center justify-center ${t.accent.primaryBg}">
  <Bot className="w-3 h-3 ${t.accent.primary}" />
</div>

// ChatInterface 消息气泡: 24x24 容器 + 12x12 图标
<div className="w-6 h-6 rounded-md flex items-center justify-center bg-indigo-500/20">
  <Bot className="w-3 h-3 text-indigo-400" />
</div>
```

### 7.2 文件扩展名 → 图标映射表

```typescript
// FileManager 中的映射逻辑
const getFileIcon = (filename: string) => {
  const ext = filename.split('.').pop()?.toLowerCase();
  switch (ext) {
    case 'tsx':
    case 'jsx':
    case 'ts':
    case 'js':
      return { icon: FileCode2, color: t.fileIcon.tsx };
    case 'json':
      return { icon: FileJson, color: t.fileIcon.json };
    case 'css':
      return { icon: Paintbrush, color: t.fileIcon.css };
    case 'md':
      return { icon: FileText, color: t.fileIcon.md };
    case 'svg':
      return { icon: ImageIcon, color: t.fileIcon.svg };
    case 'png':
    case 'jpg':
    case 'gif':
      return { icon: ImageIcon, color: t.fileIcon.svg };
    case 'd.ts':
      return { icon: FileType, color: t.fileIcon.ts };
    default:
      return { icon: File, color: t.fileIcon.default };
  }
};
```

## 10. 无障碍规范

### 10.1 ARIA 标签

每个图标按钮必须提供 `aria-label` 和 `title` 属性:

```typescript
// 正确 ✅
<button
  aria-label="通知中心"
  title="通知中心"
  className={`p-1.5 rounded-lg ${t.interactive.headerBtn}`}
>
  <Bell className="w-4 h-4" />
</button>

// 错误 ❌ — 无辅助标签
<button className="...">
  <Bell className="w-4 h-4" />
</button>
```

### 10.2 装饰性图标

非交互式的装饰性图标应标记为 `aria-hidden`:

```typescript
// 纯装饰 (如消息头像中的图标)
<Bot className="w-3 h-3" aria-hidden="true" />
```

### 10.3 屏幕阅读器

- 所有按钮的 `aria-label` 使用中文名称
- `title` 与 `aria-label` 保持一致
- 禁用按钮添加 `aria-disabled="true"`

### 10.4 焦点可见性

## 12. 性能优化

### 12.1 Tree-shaking

```typescript
// ✅ 按需导入 — 仅打包使用的图标
import { Send, Plus, Code } from 'lucide-react';

// ❌ 全量导入 — 打包所有图标 (数百个)
import * as Icons from 'lucide-react';
```

Lucide React 的 ESM 结构支持 Vite/Rollup 自动 tree-shaking，每个图标约 ~1-2KB (gzip 后更小)。

### 12.2 SVG 内联

Lucide 图标在运行时渲染为内联 SVG，无需额外 HTTP 请求。

### 12.3 重渲染控制

图标组件本身是无状态的，依赖父级 `className` 和 `style` props 驱动样式变更。确保:

- 父级按钮使用 `useCallback` 稳定事件引用
- 图标 `className` 字符串在主题不变时保持稳定（由 token 驱动）

## 13. 命名规范 (规划)

### 13.1 命名规则

```
{category}-{name}-{variant}-{size}
```

| 参数       | 说明     | 示例                                                                |
| ---------- | -------- | ------------------------------------------------------------------- |
| `category` | 图标分类 | nav, func, view, ai, user, status, device, term, file, theme, infra |
| `name`     | 图标名称 | home, back, preview, code, search, more                             |
| `variant`  | 变体     | outlined (默认)                                                     |
| `size`     | 尺寸     | 12, 14, 16, 18, 20                                                  |

**示例**: `nav-back-outlined-16` = 导航类返回图标，线性，16px

### 13.2 当前实际命名

YYC3 直接使用 Lucide 原始组件名（PascalCase）:

```typescript
// 直接使用 Lucide 导出名
<ChevronLeft />
<AlertCircle />
<FileCode2 />
```

无自定义包装层，保持最小抽象开销。
