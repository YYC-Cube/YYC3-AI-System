## 🔧 技术实现细节

### 1. Playwright/Vitest 冲突解决方案

**问题根源:**
- Vitest 和 Playwright 都使用全局匹配器（matchers）
- 两个框架同时加载时会导致 `Symbol($$jest-matchers-object)` 冲突

**解决方案:**
```typescript
// playwright.config.ts - 独立配置
export default defineConfig({
  testDir: './e2e',  // 关键：独立目录
  // ... 其他配置
})
```

**目录结构:**
```
project/
├── src/app/components/__tests__/  # Vitest 单元测试
│   ├── *.test.ts
│   └── *.test.tsx
├── e2e/                           # Playwright E2E 测试
│   ├── *.e2e.ts
│   └── sample.e2e.ts
└── playwright.config.ts           # Playwright 配置
```
