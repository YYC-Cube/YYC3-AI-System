## 各分类详细说明

### 01-团队规范

包含团队开发所需的所有规范和标准文档。

**子目录：**
- `01-开发规范/` - 代码编写规范、开发流程、质量标准
- `02-文档规范/` - 文档编写格式、标头规范、版本管理
- `03-命名规范/` - 文件命名、变量命名、函数命名规范

**关键文档：**
- [YYC3-团队规范-开发标准.md](01-团队规范/01-开发规范/YYC3-团队规范-开发标准.md)
- [Guidelines.md](01-团队规范/01-开发规范/Guidelines.md)

### 02-项目规划

**子目录：**

## 文档规范

所有文档都遵循 YYC³ 团队规范，包括：

1. **标头规范** - 包含文件信息、作者、版本、更新日期等
2. **格式规范** - 使用 Markdown 格式，统一的标题层级
3. **命名规范** - 使用 kebab-case 命名，清晰描述文档内容
4. **版本管理** - 语义化版本号，记录更新历史

## 文档维护

- **更新频率** - 根据项目进展及时更新
- **审核流程** - 重要文档需要团队审核
- **版本控制** - 使用 Git 进行版本管理
- **备份策略** - 定期备份重要文档
