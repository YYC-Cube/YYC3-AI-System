## 📂 代码文档

### 源代码文件组织

所有源代码文件遵循以下组织结构：

```
src/
├── app/
│   ├── components/          # UI组件
│   │   ├── ChatInterface.tsx
│   │   ├── ConflictResolutionDialog.tsx
│   │   ├── FileManagerVirtual.tsx
│   │   ├── MessageListVirtual.tsx
│   │   ├── OfflineStatusIndicator.tsx
│   │   ├── PerformanceMonitor.tsx
│   │   ├── ServiceWorkerRegister.tsx
│   │   ├── SyncStatusPanel.tsx
│   │   └── WebSocketStatusPanel.tsx
│   └── ...                 # 其他组件
├── config/                # 配置文件
│   ├── precache-manifest.ts
│   └── ...                # 其他配置
├── services/               # 服务层
│   ├── sync-queue-service.ts
│   ├── sync-manager-service.ts
│   ├── conflict-resolution-service.ts
│   ├── offline-degradation-service.ts
│   ├── websocket-service.ts
│   ├── performance-monitor-service.ts
│   ├── api-cache-service.ts
│   ├── cache-strategy-service.ts
│   └── ...                # 其他服务
├── types/                 # 类型定义
│   ├── sync.ts
│   ├── offline.ts
│   ├── websocket.ts
│   └── ...                # 其他类型
└── ...                     # 其他源代码
```

### 代码标头规范

所有代码文件必须包含以下标头：

```typescript
/**
 * @file {FILE_NAME}
 * @description {FILE_DESCRIPTION}
 * @author {AUTHOR_NAME} <{AUTHOR_EMAIL}>
 * @version {VERSION}
 * @created {CREATE_DATE}
 * @updated {UPDATE_DATE}
 * @status {STATUS}
 * @license {LICENSE_TYPE}
 * @copyright Copyright (c) {YEAR} YanYuCloudCube Team
 * @tags {TAGS}
 */
```

详见: `01-YYC3-团队规范-代码标头.md`

## 📊 文档状态管理

### 文档生命周期

| 状态 | 说明 | 使用场景 |
|------|------|----------|
| draft | 草稿 | 初始编写阶段，内容不完整 |
| dev | 开发中 | 内容编写中，可能频繁变更 |
| review | 审核中 | 内容完成，等待审核 |
| stable | 稳定 | 内容稳定，可供参考 |
| active | 活跃 | 文档正在使用和更新 |
| deprecated | 已废弃 | 不再维护，将被替换或删除 |

### 版本管理

遵循语义化版本号 (SemVer) 规范：`MAJOR.MINOR.PATCH`

- **MAJOR (主版本号)**: 文档结构重大变更，不兼容旧版本
- **MINOR (次版本号)**: 新增内容，向下兼容
- **PATCH (修订号)**: 内容修正，向下兼容
